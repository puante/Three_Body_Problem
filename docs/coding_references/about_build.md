# 빌드를 그래서 어떻게 하는가?

우선 가장 기본적으로 써야 하는 모듈은 pyinstaller이다. 그런데 이 pyinstaller에는 상당히 짜증나는 점이 존재하는데, 바로 모듈에 관한 내용이다. 대체 왜 이렇게 해 둔건지, 또는 기술상의 이유로 불가능한 것인지 또는 내 착각인지는 모르겠으나 이상할 정도로 **onefile로 만들어두면 paths, hidden import 등 기본적으로 내장되어 있는 다양한 옵션을 사용하더라도 외부의 모듈을 가져오지 못하는 아주 짜증나는 버그가 지속적으로 발생하였고 이 때문에 나는 거의 *총합 4시간 가량* 을 날려먹었다.** 이러한 잘못된 정도를 전달해준 클로드에게 정말 세네번은 더 절을 하고 싶다.

우선 이 프로그램을  빌드하기 위한 작업은 다음과 같다:
1. 우선 돌릴 파일을 준비한다.
2. pyinstaller 명령어를 돌린다.

```pyinstaller --noconsole --onefile "_파일 경로"```

위의 명령어는 pyinstaller를 사용 할 때 가장 기본적으로 사용되는 명령어이다. 본인은 위의 내용에서 gui 메인 파일을 빌드할 때에는 --onefile을 사용하였다. 기본적으로 python 모듈을 로드하는 것이 아닌 자신이 현재 위치하고 있는 폴더의 **exe 파일을 실행**시키는 방법이기 때문에 문제가 없으나 screen_show를 빌드할 때에는 onefile로 만드니 자꾸 py 파일을 인식하지 못하여 onefile 옵션을 제거하였다.

현재까지 확인된 내용은 다음과 같다:
1. 그냥 모듈은 제발 _internal에 생으로 넣어버리자.

paths니 hidden import니 뭐니 이상한 옵션 넣을 바에** 상남자처럼 _internal 폴더에 넣는 것**이 낫다. 몇시간의 금같은 시간을 직접 돌 갈듯이 갈아본 결과 알아낸 것은 _internal 폴더에 생으로 넣는게 효율적이다. 본인은 코드에 언급되는 파일들이 이름 앞자리만 같을 뿐 package로 묶여 있지 않기 때문에 그냥 집어넣었으나, 만약 package로 묶여있는 개인 라이브러리 등이 코드에 있다면 그 라이브러리 폴더를 통째로 넣어주면 된다. 다른 종류의 유명한 라이브러리인 matplotlib이나 PyQt5도 유사한 형태를 띄는 것을 확인하였다. 제발 이상한 소리 듣고 나처럼 시간 날리지 않고 _internal 폴더에 넣길 바란다. 대형 프로젝트로 간다면 모르겠지만 일단 내가 직접 겪어보면서 깨달은 내용은 위의 내용이다. 
지금까지 클로드가 알려준 대로 했던 모듈을 로드하는 방법 중 **완전무결하게 작동한 것이 정말 놀랍게도, 단 한개도 존재하지 않았다!** 이정도로 썼다면 정말 분노가 느껴질 것이다. 실제로 난 지금 성대 끝에서 기어나오려 하는 육두문자를 참으며 이 장문의 글을 쓰고 있다. 제발 이상한 일 하다가 시간을 낭비하지 말길 바란다. 당신들의 시간은 나보다 훨씬 더 많이 소중하니 제발 나같은 참사를 당하지 않길 바란다..

2. pyinstaller가 어렵다면 auto py to exe를 쓰는 것도 추천한다.

생각보다 많은 사람들이 잘 모르는 하꼬 프로그램이 있다. pyinstaller 기반이지만 기존에 별에 별 옵션들이 있어 명령어가 수도 없이 길어졌던 pyinstaller에 비해 gui로 깔끔하게 나오기 때문에 추천한다. pip install auto-py-to-exe를 친 다음 콘솔에 auto-py-to-exe를 치면 알아서 실행된다. 생각보다 편리하기도 하고 이 프로그램에 아이콘을 넣는 데에도 이것을 은근히 사용하였으니 개인적으로 매우 추천한다.
